- Test: `test/authorization-matrix.test.ts > matches every repository source

- Site: `packages/itotori-db/src/repositories/terminology-candidate-repository.ts:516`.
- Cause: `requirePermission(db, actor, permissionValues.catalogRead)` is called

  and there is no `@repository-permission-gate` annotation.
- Suggested fix: either move `countTerminologyTerms` into a repository class,
  or prefix the call with the annotation `// @repository-permission-gate
TerminologyCandidateRepository.countTerminologyTerms catalogRead` (whichever
  form the rest of this file uses).

### F5. `cargo test --workspace` — `kaifuu-core` 47/190 failures (reproducible only at workspace scope)

- Reproduction: `cargo test --workspace` from clean shell reliably fails 47
  tests in `kaifuu-core` with `fixture should be readable: NotFound` at
  `crates/kaifuu-core/src/lib.rs:16957` (and the parallel symptom in
  `kaifuu-engine-fixture` at `:5456`, `:5332`, `:7514`, `:7585`).
- Reproduction confirmed twice in this audit (`02b-cargo-test.log`,
  `02d-cargo-test-workspace-retry.log`).
- Counter-evidence: same crate run in isolation (`cargo test -p kaifuu-core
--lib`) passes 190/190 three times in a row. `kaifuu-engine-fixture` likewise
  passes 64/64 three times in isolation.
- Root cause hypothesis: tests use absolute paths derived from
  `env!("CARGO_MANIFEST_DIR")` (e.g. `repo_fixture_path`, `repo_root`,
  `hello_fixture_dir`), so the read target is fixed. But under
  many-binary workspace parallelism, something is racing against those files —
  most likely another test binary in the same workspace mutates a shared
  fixture directory (e.g. `kaifuu-cli`'s integration tests writing under
  `fixtures/`), or a test process is `chdir`'ing the global CWD via a child
  process. `temp_dir(name)` in `kaifuu-engine-fixture` keys only on
  `process::id() + name`, so within a single test binary names look unique —
  the contention is across binaries.
- Suggested fix (diagnosis-only): grep `cargo test --workspace` test binaries
  for any code that (a) writes inside `fixtures/`, `crates/*/fixtures`,
  `crates/*/tests/fixtures`, or (b) calls `std::env::set_current_dir`. The
  symptom is fixture-read NotFound, so something is unlinking/replacing the
  fixture file (not just CWD-dependent). Until traced, force test-binaries to
  serialize over a shared directory, or copy fixtures into a per-test temp
  tree.

## 3. Warnings (lower priority)

### W1. `cargo deny` duplicate-version warnings

- `getrandom` — 3 entries (0.2.17, 0.3.4, 0.4.3).
- `hashbrown` — 2 entries (0.16.1, 0.17.1).
- `r-efi` — 2 entries.

  versions; otherwise downgrade the lint to `allow` for these specific crates.

### W2. PG client deprecation in test suite

- `(node:3684019) DeprecationWarning: Calling client.query() when the client is
already executing a query is deprecated and will be removed in pg@9.0. Use
async/await or an external async flow control mechanism instead.`
- Origin: emitted during `pnpm --filter @itotori/db test` runs (somewhere in
  the migrations or repository integration tests).
- Suggested fix: grep `packages/itotori-db` for back-to-back `client.query()`
  calls without awaits and convert to `await`/sequential.

### W3. Vite externalization warnings during `just hello` build

- ~15 lines of `[plugin rolldown:vite-resolve] Module "crypto"/"path"/"fs"/"events"/...
has been externalized for browser compatibility, imported by .../pg/...`.
- Affects: `apps/runtime-web-review` bundle (pulls `pg` via
  `@itotori/itotori-db` transitively).
- Suggested fix: gate `@itotori/itotori-db` so the browser build does not pull
  the `pg` server client, or mark the chunk explicitly external via the Vite
  config. Build still succeeds.

### W4. `vp run` cache miss noise

- `itotori#schema:check not cached because it modified its input` appears in
  `vp run ts:test` and `vp run ts:typecheck` outputs. Indicates a task that
  rewrites a file inside its own input set.
- Suggested fix: locate the package whose `schema:check` task mutates
  generated output and either move the output outside the input glob or mark
  the task `cache: false`.

  sibling worktrees (`agent-a010f57ef2eb4edb8`, `agent-aa49edddc50e3ca1c`,

  reproducer if it recurs. Worth confirming `vp` respects `.gitignore` end-to-end.
