## 3. Recommended fixes (ordered by impact)

Highest impact first. None of these are urgent; the repo is unusually clean.

1. **Reconcile the orphaned `binary_patch_smoke.rs` helpers.** Pick one path per item:
   either delete the four genuinely-dead helpers (`exit_code`,
   `patch_result_filename`, `output_seen_filename`, `fixture_path_for`) and the
   two stale silences (`parse`, `write_smoke_summary` no longer need `#[allow(dead_code)]`),
   or wire them through the CLI dispatch arm in `crates/kaifuu-cli/src/main.rs`. DAG node:
   **KAIFUU-207**.

2. **Make the auth-matrix DB suite fail-loud locally.** Replace `describe.skipIf` in
   `packages/itotori-db/test/authorization-matrix.test.ts:976` with the same hard
   `throw new Error("DATABASE_URL is required â€¦")` pattern used by `db-test-context.ts`
   and `repository.test.ts`. Either every DB suite is opt-in or all of them are
   required â€” pick one and apply uniformly. DAG node: **ITOTORI-202**.

3. **Tighten `deny.toml`.** Move `bans.multiple-versions` to `deny` (with an
   explicit `skip` allowlist for any unavoidable duplicates) and flip `bans.wildcards`
   to `deny`. Add a short justification block above any new `skip` entry. DAG node:
   **KAIFUU-208**.

4. **Audit the `run_golden_patch_phase` argument count.** The `#[allow(clippy::too_many_arguments)]`
   at `crates/kaifuu-core/src/lib.rs:15758` is silencing a real design signal â€” eleven
   parallel positional parameters (`phase`, `game_dir`, `work_dir`, `work_child`,
   `patch_export`, `success_details`, `patch_error_code`, `patch_expected`, â€¦) is the
   point where a `GoldenPatchPhaseArgs` struct earns its keep. DAG node:
   **KAIFUU-209**.

5. **Centralize the `QueueSqlExecutor` cast.** Replace the 13 repeated
   `db as unknown as QueueSqlExecutor` sites in
   `packages/itotori-db/src/repositories/event-queue-repository.ts` with one
   `asQueueExecutor(db)` helper. Cosmetic but reduces the surface where someone
   could silently widen the executor contract. No DAG node â€” can be folded into
   the next touch of that file.

6. **Self-document the subprocess `#[ignore]` fixtures.** Change the bare
   `#[ignore]` to `#[ignore = "subprocess fixture; invoked via harness_child_command"]`
   on the five tests in `crates/utsushi-core/src/lib.rs:4623..4682`. No behaviour
   change. No DAG node â€” drive-by edit.

7. **Convert invariant `unwrap()` clusters in `offset_map.rs` to `expect()` with
   a named invariant.** Cosmetic. No DAG node.

Items the audit explicitly recommends **keeping as-is**:

- All five `#[ignore]` subprocess fixtures (Section 2.1).
- All sixteen `unreachable!()` arms (Section 2.4) â€” they encode validated-upstream
  contracts.
- All four crate-level `#![allow(...)]` in `kaifuu-vault-source/tests/` (Section 2.3).
- The non-test `#[allow]`s in `utsushi-fixture/src/launch_adapters.rs` for the
  reserved `DisplayUnavailable` enum variant + its container (semantic-code
  surface kept live).
- Strict-tsconfig and zero-eslint-config posture as-is â€” both already represent
  the strictest reasonable defaults.
